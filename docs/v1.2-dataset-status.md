## Alfred gold set — current state

Sampler picks PDFs across 6 strata (filename heuristics), one stratum per file:

| Stratum | Target | Sampled (1st run) | Pool size in Alfred |
|---|---:|---:|---:|
| `ruled_price` (spa price lists, ruled tables) | 30 | 8 | 81 |
| `unruled_menu` (food/drink menus) | 35 | 9 | 504 |
| `unruled_factsheet` (key/value factsheets) | 25 | 7 | 218 |
| `multi_page` (brochures w/ continued tables) | 15 | 4 | 41 |
| `rotated` (landscape rate cards) | 8 | 1 | 1 |
| `tagged_pdf` (accessibility-tagged) | 7 | 2 | 13 |
| **TOTAL** | **120** | **31** | 858 |

**Sampler notes**:
- 31 PDFs from 26 hotels in the first run (seed `0x12FAB1E5`). The conservative quartile cap on per-stratum pull means the next run with a higher target will fill in.
- The `rotated` filename heuristic is too narrow (1 PDF in pool). Rotation needs to be detected by parsing the PDF page rotation field, not by filename. v1.2.W1 follow-up: extend the sampler to read `/Rotate` from page dicts.
- The `tagged_pdf` heuristic catches "accessible" / "tagged" / "ada" / "wcag" — but most accessibility-tagged PDFs in the wild don't advertise the fact in their filename. The Cheval Blanc Paris LVMH RSE accessible report does (caught it). v1.2.W1 follow-up: also detect `/StructTreeRoot` presence via `pdf.zig info`.

**To bring the count toward 50 PDFs / 120 tables**:
```bash
python3 audit/v1_2_alfred_sample.py --target-pdfs 60          # bump cap
python3 audit/v1_2_annotate.py --status                       # show progress
python3 audit/v1_2_annotate.py                                # work next pending
```

The annotation harness uses `qlmanage` (macOS QuickLook) to display the PDF and `pdf.zig extract --output md -p N` to show the extractor's current output for cross-reference. Each annotated table records the structural shape (rows, cols, header_cells, row_labels) — text cell content is auto-derived at evaluation time from the extractor output, so the annotator only commits to **what the table SHOULD have**, not "verbatim what every cell says".

## Sizing rationale

Per Codex deep-research finding #4: "below ~75 gold tables, TEDS will swing too much on a handful of nasty merged-header docs". 120 gives:

- Wilson 95% CI half-width ≈ **4.5 pp** on overall TEDS-Struct
- Per-stratum n=8–35 → directional reporting (per-stratum CIs wider; treated as flags, not gates)

That's the same sample-design discipline established in Week 7 (per-tier acceptance gating, hard gate on the aggregate, soft flag on the strata).

## Annotator workflow (for the human)

```bash
# 1. (Optional) refresh skeleton list
python3 audit/v1_2_alfred_sample.py --target-pdfs 60

# 2. See what's pending
python3 audit/v1_2_annotate.py --status

# 3. Annotate the next pending PDF
python3 audit/v1_2_annotate.py
#   - Press 'y' to open the PDF in QuickLook
#   - 's' to display pdf.zig's current extraction
#   - 's 3' to display page 3 only
#   - 'a' to add a table (page, n_rows, n_cols, header_rows)
#   - 'd' to drop the last added
#   - 'c' to commit (status: done)
#   - 'q' to leave as in_progress

# 4. After a few PDFs, sanity check
python3 audit/v1_2_annotate.py --status
```

Each PDF takes ~2–5 minutes for an annotator who's seen the format. 50 PDFs × 3 min = **2.5 hours** of focused work to complete the gold set.

## What's not in the dataset (deliberately)

- **Image-only tables** (NG4) — out of v1.2 scope per the design. `pdf.zig` will abstain with `quality_flag:"image_table"` → v1.3 OCR shell-out.
- **Hand-drawn / camera-photo tables** — same NG4 bucket.
- **Encrypted PDFs** (NG5) — pdf.zig detects + skips; the gold set explicitly excludes them.
- **Excel-converted PDFs with one glyph per draw op** (CSV-origin per Codex risk #1) — included as a stretch fixture once the annotator hits one; tracked separately in `audit/tables-gold/_csv-origin.tables.json` (not yet created).
